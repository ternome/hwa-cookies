# PLAN — мобильный прелоадер-лендинг HeroWarsAlliance

## Зачем
Рекламу крутим через плееблы. Плеебл грузится ~5 секунд — это видно пользователю.
Идея: лёгкий лендинг-прелоадер играет короткое видео (стартовый этап плеебла), а пока
оно играет — настоящий плеебл грузится в фоне. По клику плеебл уже готов → ожидание
прячется «под видео».

## Флоу (по раскадровке)
1. Видео проигрывается (стартовый этап плеебла: герой + красный босс у пещеры).
2. Видео доигрывает → замирает на последнем кадре → затемнение + две карточки
   **FIRE** / **SWORD** + «Choose Your Weapon» (бейдж «Best choice» на мече).
3. Клик по любой карточке → плеебл.

Целевой плеебл:
`https://heroesweb-a.akamaihd.net/i/playables/playables/hw/pldungeonroof23a.html`

## Подтверждённые решения

| # | Решение | Выбор |
|---|---------|-------|
| 1 | Обработка клика | **Предзагрузка в скрытом iframe + разворот по клику** (прячет 5 сек) |
| 2 | Медиа старта | **Видео MP4 (H.264) + poster**, не GIF |
| 3 | Исходник видео | **Запись экрана** → сожму в оптимизированный MP4 + сгенерю poster |
| 4 | Трекинг/аналитика | **Не нужен** (переход без параметров) |
| 5 | Стек | **Один self-contained `index.html`, vanilla** (inline CSS/JS) |
| 6 | URL после клика | **Не меняется** — плеебл крутится встроенным (ради мгновенности) |
| 7 | Skip видео | **Нет** — видео доигрывает, тянет плеебл |

### Проверка встраивания (пройдена)
`curl -sI` по плееблу: `HTTP/2 200`, `content-type: text/html`,
`access-control-allow-origin: *`. **Нет** `X-Frame-Options` и `CSP frame-ancestors`
→ плеебл можно держать в iframe и развернуть. Рекомендованная архитектура рабочая.

## Состояния (4 фазы)
1. **BOOT (0 мс)** — мгновенно показываем poster (первый кадр), ноль пустого экрана.
   Параллельно: видео буферизуется и автостартует (`muted` + `playsinline` + `autoplay`);
   скрытый iframe начинает грузить плеебл.
2. **PLAYING** — видео играет поверх poster. Плеебл грузится в фоне
   (iframe: `opacity:0`, `pointer-events:none`, под видео). Ловим `iframe.load` →
   флаг `playableReady`.
3. **CHOICE** — видео `ended` → замирает на последнем кадре → плавно проявляется
   затемнение + карточки FIRE/SWORD + «Choose Your Weapon». Scale-in анимация карточек.
4. **REVEAL** — тап по карточке → прячем видео+оверлей, разворачиваем iframe на весь
   экран (поверх всего). Грузился под видео → открывается мгновенно. Тап = жест →
   у плеебла разблокируется звук.

## Технические решения
- **Autoplay:** `muted playsinline autoplay preload="auto"`, MP4 с `-movflags +faststart`
  (moov в начало → стримится). Если `video.play()` отклонён (low-power/data-saver) —
  фолбэк-таймер на длительность ролика всё равно выводит CHOICE, чтобы воронка не зависла.
- **Тайминг предзагрузки:** запуск загрузки iframe по событию видео `playing`
  (через миг после старта) — видео не конкурирует за полосу на старте, плеебл всё равно
  получает почти весь head-start. Тюнится.
- **Плеебл не успел догрузиться к клику:** на REVEAL — крошечный спиннер внутри, пока
  не сработает `iframe.load`. Всё равно сильно быстрее текущих 5 сек.
- **Замирание на последнем кадре:** `loop=false`, видео-элемент сам держит последний
  кадр; подстраховка — last-frame картинка при необходимости.
- **Размеры:** портрет, `object-fit:cover` для видео, full-bleed iframe, safe-area insets.
- **Вес видео:** цель ≤ ~1.5 МБ, чтобы первые кадры приходили почти мгновенно.

## Ассеты (получены, проанализированы)

### `preload-cast.mov` — исходник видео
- Факты: **856×1858, 120 fps, 5.04 c, 14.7 МБ, ~23 Мбит/с** (сырьё со скринкаста).
- Таймлайн: 0–2.8с сцена с боссом у пещеры (хук) → ~2.8–3.0с панорама →
  3.0–3.9с чистый вид карты → **с ~3.65с появляется CTA плеебла «PLAY NOW»** (мешает).
- **Точка обрезки: 3.6 c** — последний чистый кадр карты (без баннера), на нём замираем
  и кладём оверлей. Совпадает с кадром 2 раскадровки по смыслу.
  - Нюанс: в раскадровке на кадре-заморозке виден логотип HeroWars + кнопка INSTALL
    самого плеебла; на кадре 3.6с их нет (чище под наш оверлей). Если нужен брендинг —
    пересмотрим точку. **По умолчанию: чистый кадр 3.6с.**

### Рецепт обработки видео
```
ffmpeg -y -i preload-cast.mov -t 3.6 \
  -vf "fps=30,scale=720:-2:flags=lanczos" \
  -c:v libx264 -profile:v main -pix_fmt yuv420p -crf 27 -preset slow \
  -an -movflags +faststart preloader.mp4
```
- 120→30 fps, ширина 720 (ретина-достаточно, режет вес), CRF 27, без звука (`-an`),
  faststart. Цель ≤ ~1.5 МБ (ожидаю меньше). При необходимости подкрутим CRF/ширину.
- Poster (мгновенный фон в BOOT) = первый кадр: `scale=720:-2` → `poster.webp` (q~80).

### Карточки `FIRE.png` / `SWORD.png`
- Факты: 835×1354 (737 КБ) и 779×1352 (835 КБ) — самодостаточные карты со своим фоном.
- План: ресайз до ~360px ширины + WebP с альфой → ~30–60 КБ каждая. Углы/тень — CSS.
- **Бейдж «Best choice»** на карточках не запечён — добавлю отдельным элементом поверх
  меча (зелёный pill + 👍, как в игре/раскадровке).
